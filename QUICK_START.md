# 🚀 Быстрый старт KETEM EDU

Полная инструкция по запуску приложения с функционалом реального времени.

## 📁 Структура проекта

```
ketem-edu/
├── backend/              # Socket.IO сервер
│   ├── server.js        # Основной файл сервера
│   ├── package.json     # Зависимости backend
│   └── README.md        # Документация backend
├── src/                  # Frontend React приложение
├── .env                  # Настройки frontend
└── package.json         # Зависимости frontend
```

## ⚡ Запуск приложения

### Шаг 1: Запустите Backend (Socket.IO сервер)

Откройте **первый терминал**:

```bash
# Перейдите в папку backend
cd backend

# Установите зависимости (только первый раз)
npm install

# Запустите сервер
npm start
```

Вы должны увидеть:
```
🚀 Socket.IO server running on http://localhost:3000
📡 WebSocket endpoint: ws://localhost:3000
```

### Шаг 2: Запустите Frontend (React приложение)

Откройте **второй терминал** (не закрывая первый):

```bash
# Вернитесь в корневую папку проекта
cd ..

# Запустите React приложение (Vite уже должен быть запущен)
# Если не запущен, используйте команду вашего билдера
```

### Шаг 3: Откройте браузер

Перейдите на `http://localhost:5173` (или ваш порт Vite)

## 🧪 Тестирование функционала

### Тест 1: Проверка подключения

1. Войдите в систему:
   - **Студент**: `student@study.com` / `student123`
   - **Админ**: `admin@study.com` / `admin123`

2. В консоли backend должно появиться:
   ```
   ✅ User connected: xyz789
   👤 Студент (student) подключился
   ```

### Тест 2: Чат в реальном времени

1. Откройте **два окна браузера** (или вкладки в режиме инкогнито)
2. В первом окне войдите как **студент**
3. Во втором окне войдите как **админ**
4. В обоих окнах нажмите на оранжевую кнопку чата (💬) в правом нижнем углу
5. Отправьте сообщение из любого окна
6. ✅ Сообщение должно появиться в обоих окнах мгновенно!

### Тест 3: Онлайн пользователи

1. В личном кабинете студента прокрутите вниз
2. Найдите блок "Онлайн пользователи"
3. ✅ Вы должны увидеть всех подключенных пользователей с зелеными статусами

### Тест 4: Индикатор "печатает"

1. В окне чата начните вводить текст (не отправляя)
2. ✅ В другом окне должно появиться: "Студент печатает..."

## 🔧 Решение проблем

### Backend не запускается

```bash
cd backend
rm -rf node_modules package-lock.json
npm install
npm start
```

### CORS ошибки в консоли браузера

Проверьте, что в `backend/server.js` указан правильный URL:
```javascript
origin: 'http://localhost:5173', // Должен совпадать с вашим frontend
```

### Чат не работает

1. ✅ Проверьте, запущен ли backend (`npm start` в папке `backend`)
2. ✅ Проверьте консоль браузера на ошибки
3. ✅ Проверьте консоль backend на логи подключения
4. ✅ Убедитесь, что в `.env` правильный `VITE_SOCKET_URL=http://localhost:3000`

### "Connection failed" в консоли

Backend не запущен или недоступен. Запустите:
```bash
cd backend
npm start
```

## 🚀 Production Deploy

### Backend (Node.js)

Деплой на любой Node.js хостинг:
- Heroku
- Railway
- Render
- DigitalOcean
- AWS

Обновите `VITE_SOCKET_URL` в `.env`:
```env
VITE_SOCKET_URL=https://your-backend.herokuapp.com
```

### Frontend (React)

Build и deploy на:
- Vercel
- Netlify
- Cloudflare Pages

Обновите CORS в `backend/server.js`:
```javascript
origin: 'https://your-frontend-domain.com'
```
